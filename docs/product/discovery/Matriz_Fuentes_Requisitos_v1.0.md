Prioridad: `P0` bloqueante, `P1` MVP/corte inicial, `P2` posterior. Estados usados únicamente: confirmado, parcialmente confirmado, contradictorio, supuesto, pendiente, descartado.

> Todas las filas incluyen fuente. Una fuente de stakeholder acredita que el requisito fue solicitado, no que sea legal, contractual o técnicamente aprobado.

## Comprobaciones de cobertura

- Los dominios solicitados están cubiertos: multiempresa, IAM, CRM, cotización/venta/cobranza/comisiones, inventario/custodia/préstamos/cortes, obras/cuadrillas/asistencia/GPS/fotos/documentos laborales, procedimientos/calidad/ITP/NCR/CAPA/RFI, activos/EPP/SST, viáticos/costos/rentabilidad/informes/facturación/contabilidad/migración, auditoría/seguridad/offline/IA.
- Ninguna fila carece de fuente; las filas pendientes se mantienen como requisitos solicitados que requieren decisión, no como reglas aprobadas.
- El acuerdo comercial aparece solo como contrato/política específica. El Reglamento aparece como política interna de autoridad baja sujeta a validación.
